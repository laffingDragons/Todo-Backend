Events documentation

Events to Listen

verifyUser event

This event ("verifyUser") has to be listened on the user's end to verify user authentication. user will only be set as online user after verification of authentication token.

online-user-list event

This event ("online-user-list") has to be listened on the user's end to identify all available users that are currently online.


userId event

This event's name is actually dynamic and refers to the userId of the logged in user.

This event ("userId") has to be listened to identify an individual notify message that has been received. Example output it as follows:

            {
                notifyId: 'unique notify id',
                message: 'you message',
                createdOn: date, 
                receiverId: 'userId of receiver',
                receiverName: full name of receiver,
                senderId: 'userId of sender',
                senderName: 'full name of sender'
            }

task-changes event

This event ("task-changes") has to be listen when there is changes in task like create, edit, delete or undo a task. The following data has to be emitted

            {
              senderName: 'John Doe',
              senderId: 'userId of sender',
              message: 'your chat message',
              createdOn: date
              }

auth-error event

This event ("auth-error") has to be listened to identify an error which will occur when the data is not sent properly in the events to be emitted. Example output it as follows:

                    {
                      status: 500,
                      error: 'error message'
                  }

Events to emit

set-user event

This event ("set-user") has to be emitted when a user comes online. User can only be set as online only after verification of authentication token. Which you have pass here. The following data has to be emitted authentication token

notify event

This event ("notify") is emitted when there are user requests for adding or requesting friends request. The following data has to be emitted

            {
                senderName: 'John Doe',
                senderId: 'userId of sender',
                message: 'your chat message',
                receiverId: 'userId of receiver',
                receiverName: full name of receiver,
                createdOn: date
                }

task-notify event

This event ("task-notify") is emitted when there are changes in the task like create, edit, delete or undo. The following data has to be emitted

            {
                senderName: 'John Doe',
                senderId: 'userId of sender',
                message: 'your chat message',
                createdOn: date
                }

disconnect event (optional)

This event ("disconnect") has to be listened, when the user is disconnected to show proper message.
