Events documentation

Events to Listen

verifyUser event

This event ("verifyUser") has to be listened on the user's end to verify user authentication. user will only be set as online user after verification of authentication token.

online-user-list event

This event ("online-user-list") has to be listened on the user's end to identify all available users that are currently online. All available users can be shown to the user based on which he can start chatting with a user. The output will be an object, object has key as userId and value as full name.


userId event

This event's name is actually dynamic and refers to the userId of the logged in user.

This event ("userId") has to be listened to identify an individual chat message that has been received. Example output it as follows:

            {
                chatId: 'unique chat id',
                message: 'you message',
                createdOn: date, 
                receiverId: 'userId of receiver',
                receiverName: full name of receiver,
                senderId: 'userId of sender',
                senderName: 'full name of sender'
            }

room-msg event

This event ("room-msg") has to be listened to identify a chatroom message that has been received. Example output it as follows:

            {
                  chatId: 'unique chat id',
                  message: 'you message',
                  createdOn: date, 
                  chatRoom: 'chatroom Id',
                  senderId: 'userId of sender',
                  senderName: 'full name of sender'
              }

typing-user event

This event ("typing-user") has to be listened to identify the user chating with is typing. Example output it as follows:

              {
                userId: 'userId',
                name: 'First name of user'
              }

error-occurred event

This event ("error-occurred") has to be listened to identify an error which will occur when the data is not sent properly in the events to be emitted. Example output it as follows:

                    {
                      status: 123,
                      error: 'error message'
                  }

disconnect event (optional)

This event ("disconnect") has to be listened, when the user is disconnected to show proper message.


Events to emit

set-user event

This event ("set-user") has to be emitted when a user comes online. User can only be set as online only after verification of authentication token. Which you have pass here. The following data has to be emitted authentication token

chat-msg event

This event ("chat-msg") has to be emitted to identify an individual chat message that has been send. The following data has to be emitted

            {
                senderName: 'John Doe',
                senderId: 'userId of sender',
                receiverName: 'Smith',
                receiverId: 'userId of receiver',
                message: 'your chat message',
                createdOn: date
                }

chatroom-msg event

This event ("chatroom-msg") has to be emitted to identify a chatroom chat message that has been send. The following data has to be emitted

                {
                  senderName: 'John Doe',
                  senderId: 'userId of sender',
                  message: 'your chat message',
                  chatRoom: 'chatroom Id',
                  createdOn: date
                }

subscribe-room event

This event ("subscribe-room") has to be emitted to join a chatroom . The following data has to be emitted

            {
             chatRoom: 'chatroom Id',
            }


create-room event

This event ("create-room") has to be emitted to create a new chatroom . The following data has to be emitted

                {
                  roomId: chatroom Id,
                  roomName : chatroom name,
                  members: array of userId that will be in chatroom,
                  admin: userId of the person who creates group,
                }

typing event

This event ("typing") has to be emitted when user start typing . The following data has to be emitted

                  {
                    userId: 'userId',
                    name: 'First name'
                  }
